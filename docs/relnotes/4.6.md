## Upgrade Notes

`timeout.cplane_operation` and `timeout.max_keepalive` now also configure static Timeout values on dedicated intra-cluster HTTP clients, set once at node startup. Hot-reload via ais config cluster set continues to update the global read-mostly values used elsewhere; the HTTP clients' timeouts require a node restart to re-apply. The CLI warns accordingly.

> Rationale: removes per-call context.WithTimeout overhead from the intra-cluster control-plane hot path - keepalive probes (continuous, lifetime of cluster) and broadcast fanout (every API path through bcastGroup, including list-objects and get-batch).
